## 5. 训练指令

### 5.1 SFT（LLaMA-Factory LoRA）

#### 前置条件

1. **基座模型**下载到 `models/Qwen2.5-VL-7B-Instruct`（`Qwen/Qwen2.5-VL-7B-Instruct`）。
2. **数据**已按 §4 预处理完成。
3. **LLaMA-Factory** 已安装（本机路径 `/root/autodl-tmp/LLaMA-Factory`）：

```bash
git clone https://github.com/hiyouga/LLaMA-Factory.git /root/autodl-tmp/LLaMA-Factory
cd /root/autodl-tmp/LLaMA-Factory && pip install -e .
```

#### 注册数据集

本项目根目录有 `dataset_info.json`，定义了 `iu_xray_report` / `iu_xray_report_val`，数据文件相对路径为 `data/sft/train.json`、`data/sft/val.json`。

**实际操作**：将上述两条配置**合并进** `LLaMA-Factory/data/dataset_info.json`（与项目根目录 `dataset_info.json` 内容一致）。训练时通过 `--dataset_dir` 指向本项目根目录，LLaMA-Factory 会读取该目录下的数据文件。

#### 启动训练

在 **LLaMA-Factory 目录**下执行（路径按本机 `PROJECT_ROOT` 调整）：

```bash
export PROJECT_ROOT=/root/autodl-tmp/chest-report-grpo
export HF_ENDPOINT=https://hf-mirror.com

cd /root/autodl-tmp/LLaMA-Factory

llamafactory-cli train \
  --stage sft \
  --model_name_or_path "$PROJECT_ROOT/models/Qwen2.5-VL-7B-Instruct" \
  --dataset_dir "$PROJECT_ROOT" \
  --dataset iu_xray_report \
  --eval_dataset iu_xray_report_val \
  --template qwen2_vl \
  --finetuning_type lora \
  --output_dir "$PROJECT_ROOT/checkpoints/qwen25vl-iu-sft-lora" \
  --per_device_train_batch_size 2 \
  --gradient_accumulation_steps 4 \
  --learning_rate 2e-4 \
  --num_train_epochs 3 \
  --max_length 2048 \
  --lora_rank 8 \
  --lora_alpha 16 \
  --freeze_vision_tower true \
  --bf16 true \
  --eval_strategy steps \
  --eval_steps 200 \
  --save_steps 200 \
  --logging_steps 10 \
  --dataloader_num_workers 4
```

#### 训练结果与 checkpoint 选择


| 项目                | 值                                           |
| ----------------- | ------------------------------------------- |
| 总步数               | 837 steps（3 epoch）                          |
| train_loss        | ≈0.264                                      |
| 最终 eval_loss      | ≈0.731                                      |
| **选用 checkpoint** | **checkpoint-800**（eval_loss≈0.733，组 A 评测用） |


LoRA 产出目录：`checkpoints/qwen25vl-iu-sft-lora/`（含 checkpoint-200/400/600/800/837 等）。

#### 组 A 评测

```bash
cd "$PROJECT_ROOT"
export CHEXBERT_CHECKPOINT="$PROJECT_ROOT/models/chexbert/chexbert.pth"

python3 scripts/eval_report.py --split test \
  --adapter checkpoints/qwen25vl-iu-sft-lora/checkpoint-800 \
  --pred-output outputs/sft_test_preds.jsonl \
  --output outputs/sft_test_metrics.json
```

### 5.2 SFT LoRA merge（GRPO 冷启动前）

EasyR1 GRPO 会在基座上**新建 LoRA**，不会自动加载 SFT adapter。需先将 SFT LoRA merge 进基座：

```bash
cd "$PROJECT_ROOT"
python3 scripts/merge_lora.py \
  --adapter checkpoints/qwen25vl-iu-sft-lora/checkpoint-800 \
  --export-dir models/Qwen2.5-VL-7B-Instruct-sft-merged
```

默认从 `checkpoint-800` merge 到 `models/Qwen2.5-VL-7B-Instruct-sft-merged`（约 16GB）。若目录已存在则跳过，可用 `--force` 覆盖。

### 5.3 GRPO smoke（IU-Xray，1 step）

```bash
cd /root/autodl-tmp/chest-report-grpo
bash scripts/run_grpo_iu_xray_smoke.sh \
  2>&1 | tee outputs/iu_xray_grpo_smoke.log
```

smoke 在正式配置基础上覆盖：小 batch、`max_prompt_length=4096`、`max_pixels=1048576`、单卡关闭 KL（`disable_kl=true`）。

### 5.4 GRPO 正式训练（组 B）

**Reward**：`R = R_format + λ₁·R_clinical + λ₂·R_consistency`，其中 **λ₁=1.0，λ₂=0.3**（见 `configs/easyr1_grpo.yaml`）。

```bash
# 清理残留进程（避免 vLLM 占显存）
pkill -f verl.trainer.main; ray stop --force; sleep 2

cd /root/autodl-tmp/chest-report-grpo
export HF_ENDPOINT=https://hf-mirror.com
export PYTORCH_CUDA_ALLOC_CONF=expandable_segments:True

# 必须通过启动脚本（已写死 λ₁=1.0、λ₂=0.3；并设置 CHEXBERT 绝对路径与 CUDA lib）
nohup bash scripts/run_grpo_iu_xray.sh \
  trainer.max_steps=70 \
  trainer.val_before_train=false \
  trainer.val_freq=-1 \
  > outputs/iu_xray_grpo_group_b.log 2>&1 &
```

主配置要点（`configs/easyr1_grpo.yaml`）：

- `model_path`：`models/Qwen2.5-VL-7B-Instruct-sft-merged`
- `worker.reward.reward_function_kwargs`：`lambda_clinical: 1.0`，`lambda_consistency: 0.3`
- `trainer.save_checkpoint_path`：`checkpoints/grpo/iu_xray_group_b`
- `trainer.save_model_only: true`（PeftModel 仅保存 LoRA adapter）
- 单 step 约 **10 min**，峰值显存约 **57 GB**（actor + ref + vLLM，KL 开启）

**checkpoint**：`checkpoints/grpo/iu_xray_group_b/global_step_70/actor/lora_adapter/`

## 8. 奖励函数

**总公式**：

```
R_overall = R_format + λ₁·R_clinical + λ₂·R_consistency
```


| 文件                      | 分量            | 说明                                                            |
| ----------------------- | ------------- | ------------------------------------------------------------- |
| `format_reward.py`      | R_format      | 解析 `<findings>` / `<impression>`；失败则 clinical/consistency 置 0 |
| `clinical_reward.py`    | R_clinical    | CheXbert impression vs GT impression 的 micro-F1（仅正类）          |
| `consistency_reward.py` | R_consistency | findings / impression 阳性标签交集比：                                |
| `easyr1_entry.py`       | 组合入口          | EasyR1 按绝对路径加载                                                |


**组 B 配置**（`configs/easyr1_grpo.yaml`）：

```yaml
worker:
  reward:
    reward_function: /root/autodl-tmp/chest-report-grpo/rewards/easyr1_entry.py:compute_score
    reward_function_kwargs:
      lambda_clinical: 1.0      # λ₁
      lambda_consistency: 0.3    # λ₂
```

**说明**：`R_format` 在 SFT 后格式率已达 100% 时，对组内 advantage 贡献接近 0，但仍作**门控**（格式解析失败则临床/一致性奖励清零）。

## 9. EasyR1 与 vLLM 0.23 兼容 patch

EasyR1 官方面向 **vLLM ~0.11**；本机 **vLLM 0.23 + torch 2.11** 需保留以下本地修改（`EasyR1/` editable 安装）：


| 文件                                                 | 问题                                    | 修复                                                                         |
| -------------------------------------------------- | ------------------------------------- | -------------------------------------------------------------------------- |
| `verl/utils/vllm_utils.py`                         | LoRA 模块路径、tensor LoRA 加载 API 变更       | 兼容 import + 按 signature 构造 kwargs                                          |
| `verl/workers/sharding_manager/fsdp_vllm.py`       | `get_tensor_model_parallel_group` 不存在 | fallback `get_tp_group().device_group`                                     |
| `verl/workers/rollout/vllm_rollout_spmd.py`        | `SamplingParams.eos_token_id` 只读      | `update_sampling_params` 跳过只读字段                                            |
| `verl/utils/checkpoint/fsdp_checkpoint_manager.py` | PeftModel + 单卡 NO_SHARD 存盘失败          | LoRA-only 保存；`summon_full_params(offload_to_cpu=False)`；NO_SHARD 跳过 summon |
| `verl/trainer/main.py`                             | Ray worker 拿不到 `CHEXBERT_CHECKPOINT`  | `runtime_env` 转发环境变量                                                       |
| `rewards/chexbert_utils.py`                        | 相对路径在 Ray worker cwd 下失效              | `resolve_checkpoint_path()` 相对项目根解析                                        |


**不建议**为兼容旧 flash-attn wheel 降级 torch 到 2.6——vLLM 0.23 依赖 torch 2.11。

## 10. 问题与解决方式

### 10.1 环境与依赖


| 问题                     | 现象                | 解决                                                                    |
| ---------------------- | ----------------- | --------------------------------------------------------------------- |
| flash-attn 与 torch 不匹配 | import 失败或 ABI 错误 | 使用 **torch2.11 + cu13 + cxx11abiTRUE** 的 2.8.3 wheel                  |
| baseline 推理 nvrtc 报错   | `libnvrtc.so` 找不到 | `infer_utils.py` / `run_grpo_iu_xray.sh` 自动 prepend `LD_LIBRARY_PATH` |
| CheXbert 加载 bert 超时    | reward 初始化卡住      | `HF_ENDPOINT=https://hf-mirror.com`；确保 `bert-base-uncased` 已在本地 cache |


### 10.2 SFT / 推理


| 问题                 | 现象             | 解决                                                  |
| ------------------ | -------------- | --------------------------------------------------- |
| 预处理相对路径            | 在非项目根目录执行失败    | `preprocess_iu_xray.py` 增加 `resolve_project_path()` |
| Baseline 中文 + 低 F1 | 未微调模型不遵循英文报告格式 | 预期行为；SFT 后格式率 100%                                  |


### 10.3 GRPO / EasyR1


| 问题                                    | 现象                                                                       | 解决                                                                                           |
| ------------------------------------- | ------------------------------------------------------------------------ | -------------------------------------------------------------------------------------------- |
| **训练集为空**                             | `num_samples=0`                                                          | 双图 token 极高 → `max_prompt_length=12288`，`max_pixels=4194304`                                 |
| 单卡 OOM（actor+ref+vLLM）                | `load_fsdp_model` OOM                                                    | actor/ref offload；`gpu_memory_utilization=0.45`；开训前 `pkill` + `ray stop` 释放残留显存              |
| **CheXbert 路径在 Ray worker 失效**        | `val_before_train` 报 `FileNotFoundError`                                 | 启动脚本将 `CHEXBERT_CHECKPOINT` 转为绝对路径；`chexbert_utils.resolve_checkpoint_path()`                |
| **checkpoint 存盘 AssertionError**      | `save_model_only=false` 时 frozen vision 不在 state_dict                    | PeftModel 路径改为仅保存 `lora_adapter/` + optimizer + extra_state                                  |
| **checkpoint 存盘 NotImplementedError** | `offload_to_cpu=True and NO_SHARD is not supported`（单卡 FSDP 自动 NO_SHARD） | `fsdp_checkpoint_manager.py`：`offload_to_cpu=False`；NO_SHARD 时直接 `get_peft_model_state_dict` |
| vLLM LoRA / 并行 / SamplingParams API   | 多种 AttributeError                                                        | 见 §9 patch 列表                                                                                |
| geo3k 冒烟                              | 多次失败                                                                     | **非必须**；已跳过，直接 IU-Xray 验证主链路                                                                 |


### 10.4 设计备忘

- **SFT → GRPO**：必须 merge SFT LoRA（`scripts/merge_lora.py`）或从 EasyR1 checkpoint resume，否则 GRPO 从基座重新初始化 LoRA。
- **训练 vs 评测解码**：GRPO rollout `temperature=1.0` 采样；`eval_report.py` 贪心解码（`do_sample=False`）。
