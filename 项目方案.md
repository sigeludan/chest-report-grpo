# 一、项目思路

## 1.1 问题出发点

现有医学影像报告生成多采用纯 SFT（交叉熵），存在两点与临床需求错位：

1. **训练目标与临床价值不一致**：交叉熵优化逐词预测，医生更关心诊断是否正确、有无编造病灶；BLEU/ROUGE 高仍可能有临床错误。
2. **证据与结论缺乏显式约束**：端到端生成 Findings + Impression，易出现「Impression 有结论、Findings 无支撑」的幻觉。

## 1.2 核心思路

采用 **SFT 冷启动 + GRPO + 可验证奖励（RLVR）** 两阶段后训练：

```
阶段一 SFT（LLaMA-Factory）
  → 学会结构化输出格式与基本医学表达

阶段二 GRPO（EasyR1）
  → 用可自动计算的奖励，把优化目标从「像训练集」拉到「临床标签更准」
```

- **SFT**：真实报告监督，建立 `<findings>…</findings><impression>…</impression>` 格式与生成能力。
- **GRPO**：每个样本采样 G 条候选，用复合奖励打分，组内相对优势更新策略；**不训练奖励模型**。

## 1.3 奖励函数设计

| 奖励分量          | 计算方式                                                     | 作用                     | 优先级        |
| ----------------- | ------------------------------------------------------------ | ------------------------ | ------------- |
| **R_format**      | 正则解析 `<findings>` / `<impression>` 标签                  | 保证可解析，后续奖励可算 | P0 必做       |
| **R_clinical**    | CheXbert(Impression_pred) vs CheXbert(Impression_gt) → micro-F1 | 优化诊断结论临床准确性   | P0 必做       |
| **R_consistency** | CheXbert 阳性标签集合：P_imp ⊆ P_find 的比例                 | 约束结论有证据支撑       | P1 有余力再做 |

**R_consistency 具体定义（降风险实现，避免复杂 NER）：**

```python
P_find  = CheXbert(findings_text) 中判为阳性的病灶集合
P_imp   = CheXbert(impression_text) 中判为阳性的病灶集合
R_consistency = |P_imp ∩ P_find| / |P_imp|   if |P_imp| > 0 else 1.0
```

**总奖励：**

```
R = R_format + λ1 * R_clinical + λ2 * R_consistency
```

建议：`λ1=1.0` 起步；`λ2` 在组 B 跑通后再启用（如 0.3~0.5）。

# 三、实验方案

## 3.1 数据准备

### 3.1.1 数据来源与格式

1. 下载 IU-Xray（HuggingFace，如 `unsloth/IU-Xray` 或学界常用整理版）。
2. 解析字段：`image`（正位 PA/AP）、`image2`（侧位，若有）、`findings`、`impression`。
3. 过滤：findings 或 impression 缺失、过短（如 < 10 字符）的样本丢弃。
4. 划分：**7 : 1 : 2**（train / val / test），`seed=42`，按**病例/样本 ID** 划分，避免泄漏。

### 3.1.2 训练样本格式

**LLaMA-Factory（SFT）— ShareGPT / multimodal 格式：**

```json
{
  "messages": [
    {
      "role": "user",
      "content": "<image><image>\n请根据上述胸片（正位及侧位，如有）生成放射学报告，严格按以下格式输出：\n<findings>...</findings>\n<impression>...</impression>"
    },
    {
      "role": "assistant",
      "content": "<findings>{gt_findings}</findings>\n<impression>{gt_impression}</impression>"
    }
  ],
  "images": ["path/to/frontal.png", "path/to/lateral.png"]
}
```

侧位缺失时只放一张图，prompt 改为「根据胸片生成…」。

**EasyR1（GRPO）— 在 SFT 格式基础上：**

- `prompt`：user 消息（含图像路径）
- 无 `ground_truth` 字段在 loss 里使用；GT 仅用于 **reward 计算**（CheXbert 对比 Impression_gt）
- 建议单独维护 `reward_gt.jsonl`：`{id, impression_gt, findings_gt}`

### 3.1.3 数据规模预期

| 划分  | 约样本数 |
| ----- | -------- |
| train | ~2,700   |
| val   | ~400     |
| test  | ~800     |

（以实际过滤后为准，写入 `data/split_stats.json` 记录。）

## 3.2 阶段一：SFT 冷启动（LLaMA-Factory）

### 3.2.1 配置

| 配置项         | 取值                                                         |
| -------------- | ------------------------------------------------------------ |
| 基座           | `Qwen/Qwen2.5-VL-7B-Instruct`                                |
| 方法           | LoRA                                                         |
| rank / alpha   | 8 / 16                                                       |
| target modules | q_proj, k_proj, v_proj, o_proj, gate_proj, up_proj, down_proj |
| 学习率         | 1e-4 ~ 2e-4                                                  |
| epoch          | 3                                                            |
| batch size     | 1~2（视显存）                                                |
| max_length     | 2048                                                         |
| 冻结           | vision encoder 冻结（默认）                                  |

### 3.2.2 训练命令（示例）

```bash
llamafactory-cli train \
  --stage sft \
  --model_name_or_path Qwen/Qwen2.5-VL-7B-Instruct \
  --dataset iu_xray_report \
  --template qwen2_vl \
  --finetuning_type lora \
  --output_dir checkpoints/qwen25vl-iu-sft-lora \
  --per_device_train_batch_size 2 \
  --gradient_accumulation_steps 4 \
  --learning_rate 2e-4 \
  --num_train_epochs 3 \
  --max_length 2048 \
  --lora_rank 8 \
  --lora_alpha 16
```

### 3.2.3 验收标准（**未通过不做 GRPO**）

| 指标                | 门槛                                    |
| ------------------- | --------------------------------------- |
| 格式遵循率（val）   | ≥ 95%（能解析出 findings + impression） |
| val loss            | 收敛、无严重过拟合                      |
| 样例人工 spot check | 10 条可读、无乱码/重复崩溃              |

格式遵循率脚本：`scripts/check_format.py`，对模型生成结果跑正则解析。

### 3.2.4 产出物

- `checkpoints/qwen25vl-iu-sft-lora/`（LoRA adapter + 合并说明）
- `outputs/sft_val_predictions.jsonl`
- `outputs/sft_val_metrics.json`（格式率、BLEU、CheXbert-F1）

## 3.3 阶段二：GRPO 强化学习（EasyR1）

### 3.3.1 环境准备

1. Clone [hiyouga/EasyR1](https://github.com/hiyouga/EasyR1)，按 README 安装依赖（verl、vLLM、flash-attn 等）。
2. 将 SFT 后的 LoRA **合并到基座**，或按 EasyR1 文档加载 LoRA checkpoint 作为 GRPO 起点。
3. 准备 IU-Xray 的 GRPO 数据脚本（参考 `examples/data_preprocess/geo3k.py` 改写为 `iu_xray_grpo.py`）。

### 3.3.2 训练配置（单卡 A800 保守版）

| 配置项         | 取值                           | 说明                           |
| -------------- | ------------------------------ | ------------------------------ |
| 算法           | GRPO                           | `algorithm.adv_estimator=grpo` |
| 起点           | SFT merged checkpoint          | 必须过 SFT 验收                |
| 微调方式       | **LoRA GRPO**                  | 全参 GRPO 单卡易 OOM           |
| 采样数 G       | **4**（稳定后试 8）            | 越大越稳、越慢                 |
| max_new_tokens | **384~512**                    | 胸片报告够用，过长易 OOM       |
| KL 惩罚        | 开启，`kl_loss_coef=0.01~0.05` | 防偏离 SFT 过远                |
| batch          | 按 EasyR1 示例微调             | 以能跑通为准                   |
| reward         | 自定义 Python 函数             | 注册到 EasyR1 reward manager   |

### 3.3.3 奖励计算流程

```
候选报告 y
  ├─ 正则解析 findings / impression
  │    └─ 失败 → R_format=0, R_clinical=0, R_consistency=0
  └─ 成功 → R_format=1
       ├─ R_clinical = F1(CheXbert(imp_pred), CheXbert(imp_gt))
       └─ R_consistency = |P_imp ∩ P_find| / |P_imp|  （组 C 启用）
```

**工程要点：**

- CheXbert 在 reward 进程内 **batch 推理**（勿每条单独 forward）。
- 解析失败给 0 分，但记录失败率，避免模型「全崩」仍被平均。
- 奖励 clip 到 [0, 1] 或做 batch 内归一化，减轻 GRPO 方差。

### 3.3.4 分步实施（降风险）

| 步骤   | 内容                                             | 目标      |
| ------ | ------------------------------------------------ | --------- |
| Step 1 | EasyR1 + Qwen2.5-VL 跑通 **geo3k demo**          | 环境 OK   |
| Step 2 | 接入 IU-Xray 数据 + **仅 R_format** dummy reward | 数据流 OK |
| Step 3 | 接入 **R_format + R_clinical**（组 B）           | 主实验    |
| Step 4 | 加入 **R_consistency**（组 C）                   | 加分项    |

### 3.3.5 验收标准

| 指标                  | 期望                         |
| --------------------- | ---------------------------- |
| GRPO 能连续训练不 NaN | 必须                         |
| train reward 均值上升 | 趋势向上即可                 |
| test CheXbert-F1      | **组 B > 组 A**              |
| 组 C vs 组 B          | 有提升更好，无提升可写进分析 |

## 3.5 评测方案

### 3.5.1 自动评测（test set）

| 指标                                      | 说明                                 | 权重     |
| ----------------------------------------- | ------------------------------------ | -------- |
| **CheXbert-F1**（Impression, 14类 micro） | 核心临床指标                         | 主       |
| CheXbert-Precision / Recall               | 补充                                 | 次       |
| 格式遵循率                                | 可解析比例                           | 工程必报 |
| BLEU-1/4、ROUGE-L                         | 语言相似度                           | 辅助     |
| Findings CheXbert-F1                      | 可选，看 Findings 是否被 RL 连带改善 | 分析用   |

评测脚本统一：`scripts/eval_report.py --checkpoint ... --split test`

### 3.5.2 轻量人工评测（实习版）

- 从 test 抽 **20~30 条**（不必 50~100）。
- 三档打分（1–5）：
  1. 临床结论是否合理
  2. Impression 阳性是否在 Findings 有支撑
  3. 是否存在明显编造
- 对比组 A / 组 B（有组 C 则加上）。

### 3.5.3 训练动态分析（面试素材）

- WandB：总 reward、R_format、R_clinical、KL、format 成功率。
- 保存 5~10 条 **组 A vs 组 B** 对比 case（含 CheXbert 标签差异）。

## 3.7 预期结论与风险应对

### 3.7.1 预期结论（简历/面试表述）

- 组 A 相对 Baseline：格式与 CheXbert-F1 明显提升（SFT 价值）。
- 组 B 相对组 A：CheXbert-F1 进一步提升（**临床对齐 RL 价值**）。
- 组 C（若做）：一致性违规率下降，或 R_consistency 与临床 F1 同向改善。

### 3.7.2 风险与应对

| 风险              | 应对                                                         |
| ----------------- | ------------------------------------------------------------ |
| 单卡 OOM          | LoRA GRPO；G=4；max_new_tokens≤512                           |
| GRPO 不稳定 / NaN | 降 lr；加大 KL；先只训 R_format                              |
| CheXbert 慢       | batch inference；reward 异步（二期）                         |
| 组 B 不优于组 A   | 检查格式率、奖励尺度、G 太小；仍可作为「复现 RL 链路」写简历 |
| IU-Xray 规模小    | 写入 limitation；不阻塞实习投递                              |
