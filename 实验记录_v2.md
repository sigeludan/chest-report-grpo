# 胸片报告生成实验记录（v2：环境复现 + 数据分层重采样）

基于 **Qwen2.5-VL-7B-Instruct** 在 **IU-Xray** 上的胸片报告生成实验。  
主方法：**SFT（LLaMA-Factory LoRA）→ merge → GRPO（EasyR1 + CheXbert 奖励）**。

**项目根目录**：`/root/autodl-tmp/chest-report-grpo`  
**关联文档**：`项目方案.md`、上一轮结果见 `实验记录.md`  
**本文重点**：环境配置（含踩坑）、数据处理（SFT「分层 + 上采样」+ GRPO **原分布**校验），以及 SFT / GRPO 重训与奖励 v2。

**记录日期**：2026-07-21

## 1. 背景与本轮动机

上一轮（组 A / 组 B）test CheXbert micro-F1 约 **0.693 → 0.722**，但深入分析 `outputs/old6_24/grpo_step70_test_preds.jsonl` 后发现：


| 指标（疾病类，不含 No Finding） | SFT test  | GRPO step70 test |
| --------------------- | --------- | ---------------- |
| 预测几乎为正常模板             | **99.8%** | **99.7%**        |
| 正确命中任意疾病阳性            | ~0        | **0**            |
| impression Top-3 模板占比 | 99.8%     | 99.5%            |


结论：

1. **模板化在 SFT 阶段已形成**，旧版 GRPO 几乎拉不回来。
2. 旧 `micro-F1` 在「全阴对全阴」时 TP=0 得 0 分（或靠 `No Finding` 虚高），正常样本 clinical 优势不足，模型更依赖格式分。
3. 本轮：**SFT** 用分层上采样（hybrid）+ `sample_weight=1.6`；**GRPO** 数据保持 **原分布**（§4.7）+ clinical reward v2（§6）。

本轮 balanced SFT test（事实）：**F1 = 73.863%**；正常样本无误报；异常样本约 **70%** 仍预测为正常；全量「预测为正常」约 **92.96%**（计算见 §5.3）。  
GRPO（clinical v2）后 test：**F1 = 78.634%**；异常 **68%（102/150）** 报出病变，**32%** 仍报正常（见 §6.7）。

## 4. 数据处理



### 4.1 原始 IU-Xray → 划分 → SFT / GRPO

脚本：`scripts/preprocess_iu_xray.py`（及必要时的 `scripts/download_iu_images.py`）。  
统计落盘：`data/split_stats.json`。

**顺序（重要）**：**先过滤有效样本并划分 train/val/test，再（本轮）只对 train 做分层上采样**。  
**不是**先平衡/上采样生成新全集，再重新划分。

#### 划分流程

```text
IU-Xray 原始报告 + 图像
        │
        ▼
 ① 过滤有效样本（preprocess）
    · 必须有 frontal 图且文件存在
    · findings / impression 文本长度 ≥ min_text_len（默认 10）
    · 有 lateral 则尽量双图，否则单图
        │
        ▼
 ② 按 uid（检查/患者 ID）划分，避免同一 uid 跨集合
    · seed = 42
    · 对 uid 列表 shuffle 后按比例切分：
        train_ratio = 0.7
        val_ratio   = 0.1
        test_ratio  = 剩余 ≈ 0.2
    · 再把样本归入对应 split
        │
        ▼
 ③ 写出两套平行数据（同一划分）
    · data/sft/{train,val,test}.json
    · data/grpo/{train,val,test}.jsonl
        │
        ▼
 ④（本轮额外）仅对 train 分层上采样 → data/sft_balanced/
    · val / test 原样拷贝，不参与上采样，保证评测可比
```



#### 本仓库实际规模


| split  | 条数       | 说明       |
| ------ | -------- | -------- |
| 有效样本合计 | **3187** | 过滤后      |
| train  | **2230** | ≈70% uid |
| val    | **318**  | ≈10% uid |
| test   | **639**  | ≈20% uid |


默认参数（`preprocess_iu_xray.py`）：`--seed 42 --train-ratio 0.7 --val-ratio 0.1`。

SFT 与 GRPO **共用同一 uid 划分**；评测统一用 `test`（或训练过程用 `val`），与旧组 A/B 可比。

**输出格式**：


| 用途   | 路径                  | 格式要点                                                                       |
| ---- | ------------------- | -------------------------------------------------------------------------- |
| SFT  | `data/sft/*.json`   | ShareGPT：`messages`（user/assistant）+ `images`                              |
| GRPO | `data/grpo/*.jsonl` | EasyR1：`problem` / `answer`(JSON：`impression_gt`,`findings_gt`) / `images` |


报告统一包在：

```text
<findings>...</findings>
<impression>...</impression>
```

prompt 为英文指令（与基座对齐）；图像路径相对项目根，训练时 `image_dir` / `dataset_dir` 指向项目根。

**注册**：项目根 `dataset_info.json` → LLaMA-Factory `--dataset_dir $PROJECT_ROOT`。

### 4.2 原始 SFT 分布问题（为何要分层）

对 `data/sft/train.json`（2230）统计（关键词近似）：


| 统计                          | 约值      |
| --------------------------- | ------- |
| impression 含疾病词             | ~21–23% |
| impression 正常模板句            | ~60%+   |
| findings 含疾病词               | ~80%+   |
| findings 有病、impression 却写正常 | 约一半量级   |


高频 impression 例如：`No acute cardiopulmonary abnormality.`（单独即可数百条）。  
CE 训练偏好短、重复的安全收尾 → 测试塌成「No acute…」模板。  
**因此需要对 train 做「按 impression 是否有病」的分层与重采样**；val/test **不重采样**，保持评测可比。

### 4.3 「分层 + 上采样」逻辑

脚本：`scripts/balance_sft_data.py`  
输出：`data/sft_balanced/`（`train.json` + `val.json`/`test.json` + `balance_stats.json`）

**重要澄清**：不是「先训一轮正常、再训一轮异常」。  
而是 **构造一份混合训练集**，使每个 epoch 里异常报告出现得更频繁，正常与异常仍打乱后一起训；并给每条写入 `sample_weight`（见 §4.4）。

```text
data/sft/train.json
        │
        ▼
 ① 抽取 assistant 的 <impression>
        │
        ▼
 ② 分层标注：正常 / 异常
    · chexbert：14 类中除 No Finding 外任一阳性 → 异常
    · keyword：疾病关键词（并排除纯 “No acute…” 误触）
    · hybrid（本轮正式采用）：CheXbert OR keyword
        │
        ├── 正常分支                    ├── 异常分支
        ▼                               ▼
 ③ 模板限频                         ④ 保留全部异常
    相同 impression 文本
    最多保留 K 条（默认 K=40）
        │                               │
        └──────────┬────────────────────┘
                   ▼
 ⑤ 上采样异常，使 abnormal_ratio ≈ r（默认 r=0.45）
    目标条数：n_abn ≈ r/(1-r) * n_norm
    不足则对异常有放回重复采样；仍偏高则下调正常数量
                   │
                   ▼
 ⑥ 写入 sample_weight（正常=1.0，异常=1.6）并 shuffle
                   │
                   ▼
      data/sft_balanced/train.json
```

**本轮正式命令**：

```bash
cd /root/autodl-tmp/chest-report-grpo
export CHEXBERT_CHECKPOINT=$PWD/models/chexbert/chexbert.pth
export HF_ENDPOINT=https://hf-mirror.com

python3 scripts/balance_sft_data.py \
  --method hybrid \
  --target-abnormal-ratio 0.45 \
  --max-normal-template 40 \
  --abnormal-weight 1.6 \
  --normal-weight 1.0 \
  --seed 42
```



### 4.4 本轮分层结果（hybrid，已落盘）

统计见 `data/sft_balanced/balance_stats.json`：


| 项               | 数值                                    |
| --------------- | ------------------------------------- |
| 输入 train        | 2230                                  |
| 判定方式            | **hybrid**（CheXbert ∨ keyword）        |
| 原始异常 / 正常       | **577 / 1653**（异常约 **25.9%**）         |
| 正常模板限频后         | 1261（`max_normal_template=40`）        |
| 输出 train        | **2293**                              |
| 输出异常 / 正常       | **1032 / 1261**                       |
| 输出异常占比          | **45.01%**                            |
| `sample_weight` | 异常 **1.6**（1032 条），正常 **1.0**（1261 条） |
| val / test      | 318 / 639，不重采样；`sample_weight` 一律 1.0 |


相对原始 train：异常占比由约 26% 提到约 45%；高频正常句被限频，减轻「No acute…」对 CE 的主导。

### 4.5 样本加权 Loss（`sample_weight`）

在分层上采样之外，对异常样本再放大梯度贡献：

L = \frac{\sum_i w_i \cdot \ell_i}{\sum_i w_i}

其中 \ell_i 为第 i 条样本在 `labels ≠ -100`（assistant 监督段）上的 **token 平均 CE**；w_i 为 `sample_weight`（正常 1.0，异常 1.6）。

**数据字段**：每条 JSON 含 `"sample_weight": 1.0 | 1.6`。

**LLaMA-Factory 经典 SFT 路径补丁**（非 v1；v1 多模态当前不可用）：


| 文件                             | 作用                                                        |
| ------------------------------ | --------------------------------------------------------- |
| `data/parser.py`               | `dataset_info` 支持 `sample_weight` 列                       |
| `data/converter.py`            | ShareGPT → `_sample_weight`                               |
| `data/processor/supervised.py` | 预处理保留 `sample_weight`                                     |
| `data/collator.py`             | batch 组装 `sample_weight` 张量（勿送入 `model(**inputs)` 前未 pop） |
| `train/sft/trainer.py`         | `CustomSeq2SeqTrainer.compute_loss` 按上式加权                 |


说明：稳定版默认无 per-sample 权重钩子，故做上述本地 patch；**不要开 packing**（packed 行与单条 w_i 不对齐）。

### 4.6 数据集注册（LLaMA-Factory）

`dataset_info.json`：

- `iu_xray_report_balanced` → `data/sft_balanced/train.json`（含 `sample_weight` 列映射）
- `iu_xray_report_balanced_val` → `data/sft_balanced/val.json`

```bash
--dataset iu_xray_report_balanced \
--eval_dataset iu_xray_report_balanced_val
```



### 4.7 GRPO 数据处理（原分布，不分层）

**原则**：GRPO **只用** `preprocess` 写出的原始 uid 划分与自然比例，**不做**异常上采样、正常模板限频、也不写 `sample_weight`。与 `data/sft_balanced/` **完全脱钩**。

#### 为何 GRPO 不跟 SFT 一起分层

| 阶段 | 数据策略 | 原因 |
| --- | --- | --- |
| SFT | `data/sft_balanced/`（上采样 + w=1.6） | CE 易被高频「No acute…」主导，需抬异常曝光 |
| GRPO | `data/grpo/` **原分布**（2230 train） | 靠 rollout + 奖励优化；保持真实比例便于与 test / 旧组 B 对比 |

公平对比看 **同一 test（639）**，不要求 GRPO train 与 SFT balanced 同分布。

#### 生成与校验流程

```text
scripts/preprocess_iu_xray.py（§4.1 ①–③）
        │
        ▼
 data/grpo/{train,val,test}.jsonl     ← 与 data/sft 同一 uid 划分
        │
        ▼
 scripts/prepare_grpo_data.py
    · 默认：核对 train 图像集合 == data/sft/train.json
    · 统计关键词近似正常/异常 → data/grpo/data_stats.json
    · --refresh：再跑 preprocess --skip-extract（不改 seed，重写 sft+grpo）
```

| 项 | 路径 / 说明 |
| --- | --- |
| 生成 | `scripts/preprocess_iu_xray.py`（与 SFT 同时写出） |
| 校验 / 统计 | `scripts/prepare_grpo_data.py` → `data/grpo/data_stats.json` |
| 训练入口 | `configs/easyr1_grpo.yaml` → `data.train_files` / `val_files` |
| 禁止 | 不要把 `data/sft_balanced/train.json` 转成 GRPO 训练文件 |

## 5. 本轮 SFT 训练方式



### 5.1 配置摘要


| 项           | 取值                                           |
| ----------- | -------------------------------------------- |
| 基座          | `models/Qwen2.5-VL-7B-Instruct`              |
| 数据          | `iu_xray_report_balanced` / val 同上           |
| 微调          | LoRA r=8, α=16，`freeze_vision_tower=true`    |
| batch       | per_device=2，grad_accum=4（有效 batch≈8）        |
| LR          | **1e-4**                                     |
| epochs      | **2**                                        |

相对上一轮组 A（3 epoch、LR 2e-4、save 200、无分层/加权）：本轮用更低 LR、更少 epoch，并显式提高异常样本频率与 loss 权重，减轻正常模板过拟合。

## 6. GRPO 奖励改进与实验 Pipeline

### 6.0 GRPO 数据入口（详见 §4.7）

GRPO **原分布**数据处理（原则、字段、统计、`prepare_grpo_data.py`、与 SFT balanced 对照）见 **§4.7**。本节只记训练入口：

| 项 | 说明 |
| --- | --- |
| 训练 / 验证 | `data/grpo/train.jsonl`（**2230**）、`val.jsonl`（318） |
| 配置 | `configs/easyr1_grpo.yaml` → `data.train_files` / `val_files` |
| 禁止 | 不用 `data/sft_balanced/` |
| 训前校验 | `python3 scripts/prepare_grpo_data.py` |

### 6.1 总公式（新旧相同外壳）

```text
R = R_format + λ₁ · R_clinical + λ₂ · R_consistency
```


| 分量              | 作用                                                                    | 默认系数     |
| --------------- | --------------------------------------------------------------------- | -------- |
| `R_format`      | `<findings>` / `<impression>` 可解析则为 1，否则 0；失败则清零 clinical/consistency | 门控       |
| `R_clinical`    | 相对 GT impression 的临床对齐（**本轮改这里**）                                     | `λ₁=1.0` |
| `R_consistency` | findings 与 impression 阳性标签一致性                                         | `λ₂=0.3` |


实现入口：`rewards/easyr1_entry.py` → `rewards/compute_reward.py`；核心打分：`rewards/chexbert_utils.py::clinical_score_v2`。

标签约定：TP/FP/FN 只在 **13 个疾病类**上统计（**排除** `No Finding`），避免「全阴对全阴」靠 No Finding 刷分。

### 6.2 新 `R_clinical` 怎么设

**分支 A — GT 正常**（疾病阳性数 = 0）

```text
若 Pred 也无疾病阳性:  R_clinical = r_normal          # 1.0
若 Pred 报了假阳性:    R_clinical = max(0, r_normal - Σ γ)
  · 普通假阳性 γ = gamma_fp = 0.25
  · 严重假阳性（气胸 / 胸腔积液 / 水肿）γ = gamma_fp_severe = 0.4
```

**分支 B — GT 有病**

```text
P = TP/(TP+FP),  R = TP/(TP+FN)
Fβ = (1+β²)·P·R / (β²·P + R)     # β = fbeta = 2 → F2
R_clinical = λ_abnormal · Fβ     # λ_abnormal = 1.5
```

yaml / CLI kwargs（`configs/easyr1_grpo.yaml`）：

```yaml
r_normal: 1.0
gamma_fp: 0.25
gamma_fp_severe: 0.4
fbeta: 2.0
lambda_abnormal: 1.5
lambda_clinical: 1.0
lambda_consistency: 0.3
```



### 6.3 为什么这样设


| 设计点                     | 原因                                                            |
| ----------------------- | ------------------------------------------------------------- |
| 正常对正常给 **1.0**          | 旧 micro-F1 在 TP=0 时直接返回 **0**，正常样本 clinical 拉不开，GRPO 易只靠格式/模板 |
| 假阳性按个数扣分                | 比再套 F1 更稳；抑制「正常片乱报病」                                          |
| 严重病假阳性更大 γ              | 气胸/积液/水肿误报临床代价更高                                              |
| 有病用 **F2（β=2）**         | 相对 F1 更重 **recall**，漏诊比多报更伤分（对应 SFT 后仍有约 70% 异常漏诊）            |
| `λ_abnormal=1.5`        | 有病样本 clinical 上限高于正常满分，组内优势更跟「报对病」走                           |
| `λ₂=0.3` 保留 consistency | 防止 findings / impression 各说各话                                 |
| format 仍 0/1 门控         | 不加大 format，避免再次塌成「只靠格式分」                                      |




### 6.4 与旧 rewards 对比


|          | 旧（组 B）                                | 新（clinical v2）           |
| -------- | ------------------------------------- | ------------------------ |
| 正常 ∪ 正常  | micro-F1 → **常为 0**（或靠 No Finding 虚高） | **R=1.0**                |
| 正常 ∪ 乱报病 | F1 低/0，区分弱                            | 按 FP 个数线性扣到 ≥0           |
| 有病       | 对称 F1                                 | **F2 × 1.5**（偏召回 + 放大）   |
| 标签空间     | 14 类含 No Finding                      | **13 疾病类**               |
| 组内信号     | 正常样本 clinical 方差小                     | 正确正常 vs 幻觉病、高召回 vs 漏诊可拉开 |


行为示例（建议超参）：


| GT    | Pred     | 旧 micro-F1 | 新 R_clinical |
| ----- | -------- | ---------- | ------------ |
| 全阴    | 全阴       | 0          | **1.0**      |
| 全阴    | 1 个普通假阳性 | ≈0         | **0.75**     |
| 有病且全对 | 全对       | ≈1         | **1.5**      |
| 有病漏诊多 | 召回低      | 中等 F1      | F2 更低        |




### 6.5 GRPO 超参数（主配置）与取值理由

数据：`data/grpo/train.jsonl` **2230** 条（与 SFT 同一 uid 划分；**不做** sft_balanced 上采样）。配置：`configs/easyr1_grpo.yaml`。


| 项                         | 取值                                  | 为什么                                       |
| ------------------------- | ----------------------------------- | ----------------------------------------- |
| `rollout_batch_size`      | **16**                              | 每 step 采 16 个 prompt；单卡显存与吞吐折中（旧实验同档）     |
| `mini_rollout_batch_size` | **8**                               | 分批生成，降低峰值显存                               |
| `worker.rollout.n`        | **4**                               | 每 prompt 采 4 条 completion，GRPO 组内比相对优势    |
| `temperature`（训练 rollout） | **1.0**                             | 提高探索，争取采到「报病」候选；评测脚本仍常贪心                  |
| `top_p`（训练）               | **1.0**                             | 与 temp=1.0 搭配，少截断                         |
| val 解码                    | temp **0.6** / top_p **0.95** / n=1 | 验证稍稳，减少纯噪声                                |
| `actor.global_batch_size` | **8**                               | 策略更新 batch；配合 micro=1 控显存                 |
| `lr`                      | **5e-7**                            | RL 相对 SFT 小很多，防冲垮冷启动                      |
| `kl_coef`                 | **0.02**                            | 轻度 KL，防偏离 SFT 太远                          |
| `freeze_vision_tower`     | true                                | 与 SFT 一致，省显存                              |
| LoRA                      | r=8, α=16                           | 与 SFT 同档；EasyR1 **新建** LoRA，故需先 merge SFT |
| `total_epochs`            | **1**                               | 先跑满一轮看曲线；可再加 epoch                        |
| `max_steps`               | null（跟 epoch）                       | 正式跑不截断；smoke 可设 `max_steps=1`             |
| `val_freq` / `save_freq`  | **50**                              | 约每 50 step 验证与存盘                          |
| `gpu_memory_utilization`  | **0.45**                            | 单卡 actor+ref+vLLM 同驻，偏保守防 OOM             |


**1 epoch ， 140 step， 18 hours（一个step 8分钟左右）**

Smoke（`scripts/run_grpo_iu_xray_smoke.sh`）会缩小 batch、`max_steps=1`、`disable_kl=true`，仅验证链路。

### 6.7 GRPO 训练后 test 结果

在 balanced SFT 冷启动 + **clinical v2** 奖励下跑完 GRPO 后，test（639）结果：


| 指标                | SFT（§5.3）     | **GRPO（本轮）**     |
| ----------------- | ------------- | ---------------- |
| CheXbert micro-F1 | 73.853%       | **75.634%**      |
| 异常样本能报出病变标签       | ~30%（45/150）  | **32%（49/150）**  |

说明 clinical v2（正常满分 + 有病偏 F2）在 GRPO 阶段对抬异常召回有效；但是倾向于对已经能够识别出标签的阳性样本进一步正确地识别出更多的病变标签，而不是在完全没识别出的阳性样本上突破。
